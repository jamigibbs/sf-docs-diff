lightning-input-locationmodule

Represents a geolocation compound field that accepts a latitude and longitude value.

Support:GA
Access:global
Abstract:false
Extensible:false

Attributes

NameTypeAccessRequiredDefaultDescription
disabledglobalIf present, the geolocation fields are disabled and users cannot interact with them.
field-level-helpglobalHelp text detailing the purpose and function of the input.
labelglobalThe label of the geolocation compound field.
latitudeglobalThe latitude value. Latitude values must be within -90 and 90.
longitudeglobalThe longitude value. Longitude values must be within -180 and 180.
read-onlyglobalIf present, the geolocations fields are read-only and cannot be edited.
requiredglobalIf present, the geolocation fields must be filled out before the form is submitted. An error message is displayed if a user interacts with the field and does not provide a value.
validityglobalRepresents the validity states that an element can be in, with respect to constraint validation.
variantglobalThe variant changes the appearance of a geolocation compound field. Accepted variants include standard, label-hidden, label-inline, and label-stacked. This value defaults to standard. Use label-hidden to hide the label but make it available to assistive technology. Use label-inline to horizontally align the label and geolocation fields. Use label-stacked to place the label above the geolocation fields.

Methods

NameArgumentsAccessDescription
blurglobalRemoves keyboard focus from the latitude and longitude fields.
checkValidityglobalReturns the valid attribute value (Boolean) on the ValidityState object.
focusglobalSets focus on the latitude field.
reportValidityglobalDisplays the error messages and returns false if the input is invalid. If the input is valid, reportValidity() clears displayed error messages and returns true.
setCustomValidityForField
NameTypeDescription
messageunknownThe string that describes the error. If message is an empty string, the error message is reset.
fieldNameunknownName of the field, which must be latitude or longitude.
globalSets a custom error message to be displayed for the latitude or longitude field when the value is submitted.
showHelpMessageIfInvalidglobalDisplays error messages on the latitude or longitude field if the coordinates are invalid.
