lightning-textareamodule

Represents a multiline text input field.

Support:GA
Access:global
Abstract:false
Extensible:false

Attributes

NameTypeAccessRequiredDefaultDescription
access-keyglobalThe keyboard shortcut for input field.
autocompleteglobalControls auto-filling of the field. Set the attribute to pass through autocomplete values to be interpreted by the browser.
disabledglobalIf present, the textarea field is disabled and users cannot interact with it.
field-level-helpglobal
labelglobalYesText that describes the textarea input field.
max-lengthglobalThe maximum number of characters allowed in the textarea.
message-when-bad-inputglobalError message to be displayed when a bad input is detected.
message-when-too-longglobalError message to be displayed when the value is too long.
message-when-too-shortglobalError message to be displayed when the value is too short.
message-when-value-missingglobalError message to be displayed when the value is missing.
min-lengthglobalThe minimum number of characters allowed in the textarea.
nameglobalSpecifies the name of an input element.
placeholderglobalText that is displayed when the field is empty, to prompt the user for a valid entry.
read-onlyglobalIf present, the textarea field is read-only and cannot be edited.
requiredglobalIf present, the textarea field must be filled out before the form can be submitted.
validityglobalRepresents the validity states of the textarea input, with respect to constraint validation.
valueglobalThe value of the textarea input, also used as the default value during init.
variantglobalThe variant changes the appearance of the textarea. Accepted variants include standard, label-hidden, label-inline, and label-stacked. This value defaults to standard. Use label-hidden to hide the label but make it available to assistive technology. Use label-inline to horizontally align the label and textarea. Use label-stacked to place the label above the textarea.

Methods

NameArgumentsAccessDescription
blurglobalRemoves focus from the textarea field.
checkValidityglobalReturns the valid attribute value (Boolean) on the ValidityState object.
focusglobalSets focus on the textarea field.
reportValidityglobalDisplays the error messages and returns false if the input is invalid. If the input is valid, reportValidity() clears displayed error messages and returns true.
setCustomValidity
NameTypeDescription
messageunknownThe string that describes the error. If message is an empty string, the error message is reset.
globalSets a custom error message to be displayed when the textarea value is submitted.
setRangeText
NameTypeDescription
replacementunknownThe string to insert.
startunknownThe 0-based index of the first character to replace.
endunknownThe 0-based index of the character after the last character to replace.
selectModeunknownA string defining how the selection should be set after the text has been replaced. Possible values: - 'select': selects the newly inserted text. - 'start': moves the selection to just before the inserted text. - 'end': moves the selection to just after the inserted text. - 'preserve': attempts to preserve the selection. This is the default.
globalReplace a range of text in textarea with a new string.
showHelpMessageIfInvalidglobalDisplays error messages on invalid fields. An invalid field fails at least one constraint validation and returns false when checkValidity() is called.
